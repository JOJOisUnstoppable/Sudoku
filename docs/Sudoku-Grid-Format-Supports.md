下面介绍一下这个程序支持读入的代码文本格式。

### 单行文本

```
040050009000300100001024030730000004000009200002800070004070080020006003900400000
```

格式很简单，就是用 81 个数字来拼凑就可以了。81 个数字就相当于盘面的每一个单元格。数字的顺序从左到右、从上到下排列，对应盘面的 81 个单元格。如果是空格的话，可以使用 `0`（阿拉伯数字 0）或 `.`（小数点）来表示。

### 单行文本（带删数和可修改数值）

```
04000900280+91+250600060800+9+5000000+240+6902005073007506+1+90500309+70000+5000200080020+51:711 113 713 317 132 337 437 141 742 143 645 146 156 456 171 473 476 181 782 183 483 185 694
```

格式写法是 `盘面:删数表`。盘面依旧使用和前文类似的表达模式，但是如果出现可修改的数字（不是已知数，但自己已经填进去了的话，这个数字左边将使用加号标记。比如 `04000900280+9` 里的 `+9` 就表示这个 9 是自己填的，和已知数 9 作区分；删数表则是用三个数字一组的形式来表示。比如 `711` 表示第 1 行第 1 列（r1c1）的数字 7 已经被删除。

### Hodoku 数独题库格式

```
:0000:x:00008+7+390+83000900770+95001+80+390070+8000+78+9+50+643500+30+8+9+71007+890030+950460+7+1+84+80+7052+6+9:::
```

这个格式是 Hodoku 软件里提供的一种完整的格式，它可以从代码上给出和表示出每一个步骤的具体表示信息。前面的 `:0000:` 表示技巧的代号，每一个实现了的技巧都会有一个全局唯一的代码（具体请参考 Hodoku 给的格式）；`x` 表示这个技巧到底用在什么数字上。比如这个盘面涉及的步骤用于一个关于 6 的 X-Wing，那么这个 `x` 就会改成 6；如果表示多个数字，将使用多个数字拼接的形式；如果不表示任何数字（就只是想表示一个普通的盘面）的话，这里就用 `x` 占位。

### Hodoku 数独题库格式（带删数和可修改数值）

| 56  2    567 | 9    138  47 | 16  48  38  |
| 1   479  679 | 58   358  47 | 69  2   358 |
| 8   49   3   | 6    2    15 | 15  49  7   |
'--------------'--------------'-------------'
```

铅笔盘是另外一种表示中间盘面的方法。不过铅笔盘比较简陋，如果一个格子有啥候选数，就都呈现出来；如果只有一个数，就无法区分了：因为我们无法确定只有一个数字的格子到底是唯一余数还是提示数信息。所以铅笔盘默认规定，只有一个数字的格子默认被处理为提示数。

### 铅笔盘（带删数和可修改数值信息）

```
+-------------------+-----------------+--------------------+
| <8>   12459  249  | *6*  <7>   259  | 1245    245   *3*  |
| 1457  1245   <6>  | 58   125   <3>  | 124578  2458  <9>  |
| 1579  <3>    279  | <4>  1259  2589 | 12578   *6*   78   |
+-------------------+-----------------+--------------------+
| *2*   <7>    *3*  | <1>  <8>   45   | 456     <9>   46   |
| <6>   458    48   | <2>  59    459  | <3>     *7*   *1*  |
| 1459  1459   49   | *3*  *6*   <7>  | 458     458   <2>  |
+-------------------+-----------------+--------------------+
| <3>   2489   <1>  | 578  25    <6>  | 24789   248   478  |
| 79    <6>    2789 | 78   <4>   *1*  | 289     *3*   <5>  |
| 47    248    <5>  | <9>  *3*   28   | 2678    <1>   4678 |

```

将铅笔盘推广一下，提示数用 `<数字>` 表示，可修改数值则使用 `*数字*` 来表示，这样可以避免对一些数字产生误会：因为只有一个数的格子我们并不知道到底是唯一余数还是已知数。

### Open-sudoku 格式

Open-sudoku 格式是安卓一个叫做 Sudoku 10000+ 程序的自带文件格式。它的代码使用 81 个三元组表达盘面的基本信息。

```

三个一组的信息里，第一个数值表示格子填入的数字，而后面两个则是表达宫线和格子具体类型的相关信息；如果是空格的话，三元组一定是 `0|0|1`；而如果已经填入了数字，则三元组一定是 `数值|0|0`。
